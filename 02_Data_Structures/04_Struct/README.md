# 04_Struct: 結構體封裝

本節重點在於探討如何運用結構體組織複合型數據，並建立物件化的解題思維。

<br>

---

<br>

## 📌 學習目標 (Objectives)
- **基礎鞏固**：熟練結構體成員變數的定義、存取與自訂排序函式 (`operator<`)。
- **效率優化**：透過結構體封裝多維數據，保持程式邏輯的一致性。
- **邊界案例**：處理結構體記憶體對齊 (Memory Alignment) 帶來的額外空間損耗。

## 💡 解題筆記 (Key Notes)
- **核心重點**：當多個變數總是作為一組出現時，使用結構體可大幅簡化排序 (`std::sort`) 與搜尋邏輯。
- **常見陷阱**：忘記為自定義的結構體重載比較運算子 (`<` 或 `==`) 導致無法使用標準演算法。
- **進階思考**：在複雜數據處理時，透過結構體封裝狀態，可配合 Priority Queue 實作更高級的演算法（如 Dijkstra）。
