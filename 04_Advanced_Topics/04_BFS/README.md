# 04_BFS: 廣度優先搜尋 (Breadth First Search)

本節探討如何利用 Queue 結構，由起點開始由近到遠「逐層」向外擴展的圖形遍歷演算法。

<br>

---

<br>

## 📌 學習目標 (Objectives)
- **基礎鞏固**：掌握基於 Queue 的非遞迴 BFS 寫法，理解層序遍歷（Level-Order）的特性。
- **效率優化**：利用 BFS「第一次搜到即為最短路徑」的特性，高效率求解無權重圖的最短步數。
- **邊界案例**：妥善處理佇列可能塞滿的狀況，並在「物件入隊時」立即標記已拜訪，避免重複入隊。

## 💡 解題筆記 (Key Notes)
- **核心重點**：BFS 的核心是「擴散」，由於它是按層走訪，因此非常適合用來解決非負權重地圖中的「最短路徑」或「最少操作步數」問題。
- **常見陷阱**：忘記在節點「`push` 入隊的當下」標記 `visited = true`，而是等到 `pop` 出隊才標記，這會導致同一個節點被重複推進 Queue 中，引發記憶體崩潰 (MLE/TLE)。
- **進階思考**：當圖形的邊權重不再是固定的 1，而是有大有小時，單純的 BFS 將不再適用，必須升級為基於 Priority Queue 的戴克斯特拉演算法 (Dijkstra)。
