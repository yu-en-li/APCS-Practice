# 01_Recursion: 遞迴演算法

本節探討如何透過函數自我呼叫，將複雜的大問題拆解為相同結構的小問題來加以解決。

<br>

<br>

## 📌 學習目標 (Objectives)
- **基礎鞏固**：掌握遞迴狀態傳遞、參數演進與基礎案例 (Base Case) 的設計。
- **效率優化**：理解遞迴深度對系統 Stack 空間的限制，避免寫出冗餘的分支。
- **邊界案例**：精確設計終止條件以防範無窮遞迴，並估算記憶體消耗。

## 💡 解題筆記 (Key Notes)
- **核心重點**：遞迴的核心是「分而治之 (Divide and Conquer)」，先定義好終止條件，再推導大問題與小問題間的關係。
- **常見陷阱**：遞迴缺乏 Base Case 會導致 Stack Overflow (RE)；此外，必須注意區域變數在每一層遞迴中所佔用的記憶體空間。
- **進階思考**：當遞迴樹中出現大量「重複計算」時（如費氏數列），可透過「記憶化 (Memoization)」記錄已知答案，這也是邁向動態規劃 (DP) 的關鍵橋樑。
