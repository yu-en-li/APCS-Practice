# 05_DP: 動態規劃 (Dynamic Programming)

本節探討如何將大問題拆解為互相重疊的子問題，並透過矩陣記錄子問題的解答，進而以空間換取時間的高階演算法。

<br>

---

<br>

## 📌 學習目標 (Objectives)
- **基礎鞏固**：學會找出問題的「最佳子結構」與動態轉移方程式（State Transition Equation）。
- **效率優化**：消除遞迴帶來的重複計算，將指數級複雜度 $O(2^N)$ 最佳化至多項式級 $O(N)$ 或 $O(N^2)$。
- **邊界案例**：精確設定 DP 陣列的初始狀態（Base Cases，如 `dp[0]`、`dp[1]`）與陣列大小。

## 💡 解題筆記 (Key Notes)
- **核心重點**：DP 的精髓在於「記憶（Memoization）」與「無後效性」，前一步做出的決策不會影響未來的狀態發展。
- **常見陷阱**：轉移方程式思維不夠嚴密，漏掉某些決策分支；或者迴圈填表的順序錯誤（例如背包問題未從後往前填表），導致子問題被提前污染。
- **進階思考**：許多 DP 問題的二維矩陣通常只會參考到「上一列」的資料，因此可以利用滾動陣列（Rolling Array）將空間複雜度從 $O(N^2)$ 壓縮到 $O(N)$。
