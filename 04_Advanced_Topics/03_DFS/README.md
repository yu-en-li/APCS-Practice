# 03_DFS: 深度優先搜尋 (Depth First Search)

本節探討如何利用 Stack 架構或遞迴，沿著路徑往深處搜尋，直到無法前進後再進行回溯的圖形與狀態空間遍歷演算法。

## 📌 學習目標 (Objectives)
- **基礎鞏固**：掌握 DFS 遞迴實作、走迷宮模擬，以及利用 `visited` 陣列標記防止死迴圈。
- **效率優化**：理解狀態空間樹的展開，學會利用剪枝 (Pruning) 提前結束不合法的分支。
- **邊界案例**：精確控制地圖邊界檢查（如 $x, y$ 是否越界）與回溯時的狀態清理。

## 💡 解題筆記 (Key Notes)
- **核心重點**：DFS 的精髓在於「一條路走到黑」，非常適合用來解決「是否有解」、「走訪所有節點」或「找出所有排列組合」的問題。
- **常見陷阱**：在進行局部狀態窮舉（如 Backtracking）時，遞迴進入下一關後，**出來時一定要把狀態還原（例如 `pop` 或將標記改回 `false`）**，否則會污染其他分支。
- **進階思考**：DFS 搜尋到的第一條路徑通常「不是」最短路徑。若要求步數最少的最短路徑，通常會優先考慮 BFS。
