# 02_Stack_Queue: 堆疊與佇列

本節探討「後進先出 (LIFO)」與「先進先出 (FIFO)」兩種最基礎且極其重要的線性資料結構與線性狀態模擬。

<br>

---

<br>

## 📌 學習目標 (Objectives)
- **基礎鞏固**：熟練 `std::stack` 與 `std::queue` 的基本操作（push, pop, top, front）。
- **效率優化**：利用其線性結構特性，以 $O(1)$ 的極高效率進行資料的存取與刪除。
- **邊界案例**：處理空容器狀態，防止在容器沒有元素時呼叫 pop 或 top 導致程式崩潰。

## 💡 解題筆記 (Key Notes)
- **核心重點**：Stack 適用於具備「對稱性」、「路徑暫存」或「回溯」特性的問題；Queue 則完美對應「排隊順序」與「緩衝機制」。
- **常見陷阱**：在讀取 `.top()`、`.front()` 或執行 `.pop()` 之前，**絕對要先使用 `.empty()` 檢查**，否則必噴 Runtime Error。
- **進階思考**：在後續的圖形搜尋中，Stack 是深度優先搜尋 (DFS) 的底層架構，而 Queue 則是廣度優先搜尋 (BFS) 的核心核心。
