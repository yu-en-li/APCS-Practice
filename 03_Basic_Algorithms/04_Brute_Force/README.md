## 💡 解題筆記 (Key Notes)
- **核心重點**：遞迴的核心是「分而治之 (Divide and Conquer)」。
- **常見陷阱**：遞迴缺乏 Base Case 或變數狀態未正確回溯（Backtracking 需要 `pop`）。
- **進階思考**：遞迴產生的重複計算可透過「記憶化 (Memoization)」轉換為 DP。
